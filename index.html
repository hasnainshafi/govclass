If Tsc command is not running Do the Foloowing steps

The error message you're encountering indicates that PowerShell script execution policies on your system prevent running scripts, including the TypeScript compiler script tsc.ps1. To resolve this, you can either change the execution policy or use a different approach to compile your TypeScript code. Here's how to resolve the issue by changing the execution policy: Open PowerShell as Administrator: Right-click on PowerShell and select "Run as administrator". Check Current Execution Policy: Run the following command to check the current execution policy: powershell Copy code

Get-ExecutionPolicy

Change Execution Policy: Run the following command to change the execution policy to allow script execution: powershell Copy code

Set-ExecutionPolicy RemoteSigned

This command allows the execution of scripts that are downloaded from the internet, but requires digitally signed scripts for locally created scripts.If you're in an environment where you can't change the execution policy, you can use the following command to allow script execution just for the current session: powershell Copy code

Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass

Confirm Change: When prompted, confirm the change by typing Y and pressing Enter. Retry Running TypeScript Compiler: After changing the execution policy, try running the TypeScript compiler (tsc) again. If you're still having issues or if you prefer an alternative approach, you can use npx to run TypeScript compiler without changing the execution policy: bash Copy code

npx tsc

This command will run tsc from your locally installed Node.js packages without requiring script execution policy changes. Choose the method that best fits your requirements and environment.
